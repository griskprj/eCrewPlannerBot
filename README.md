# 🎯 Telegram Bot для управления мероприятиями
Бот для создания, публикации и управления мероприятиями в Telegram группах. Позволяет администраторам создавать события, собирать регистрации участников, отправлять напоминания и экспортировать данные.

## 📋 Возможности

### Для администраторов
* **Управление группами** – бот автоматически добавляет группы в базу при добавлении
* **Создание мероприятий** – интерактивная форма с шагами (название, дата, время, место, описание)
* **Публикация** – вывод мероприятия в группу с кнопкой записи
* **Управление событиями:**
    * Просмотр списка мероприятий
    * Редактирование (в разработке)
    * Отмена / завершение
    * Удаление
* **Работа с участниками:**
    * Просмотр списка записавшихся
    * Экспорт в CSV
    * Отправка напоминаний в группу
    * Коммуникация – отправка произвольных сообщений в группу

### Для участников групп
* **Запись на мероприятия** – один клик по кнопке *"Записаться"*
* **Подтверждение** – участник получает уведомление в личные сообщения
* **Уведомления** – напоминания о предстоящих событиях

## 🚀 Быстрый старт
### 1. Добавление бота в группу
* Добавь бота в группу (username бота)
* Сделай бота администратором группы (нужно для отправки сообщений)
* Бот автоматически добавит группу в базу и отправит приветствие

### 2. Настройка администраторов бота
Администраторы бота указываются в файле .env через переменную ADMIN_IDS *(см. раздел Установка).*

### 3. Основные команды
* `/start` – Начало работы, главное меню
* `/menu` – Открыть главное меню

### 🛠 Технические требования
* Python 3.9 или выше
* SQLite3 (встроен в Python)
* Доступ в интернет для работы с Telegram API

### 📦 Установка

#### Локальный запуск

1. Клонируй репозиторий
```
git clone <url-репозитория>
cd bot
```

2. Создай виртуальное окружение
```
python -m venv venv
source venv/bin/activate  # для Linux/Mac
# или
venv\Scripts\activate  # для Windows
```

3. Установи зависимости
```
pip install -r requirements.txt
```

4. Создай файл конфигурации
```
cp .env.example .env
# Отредактируй .env, добавь токен бота и ID администраторов
```

5. Запусти бота
```
python bot.py
```

#### Запуск на сервере *(рекомендуется)*
**Вариант 1: Через systemd (Linux)**

1. Выполни шаги 1-4 из локальной установки на сервере

2. Создай systemd сервис:
```
sudo nano /etc/systemd/system/bot.service
```

3. Вставь конфигурацию (замени пути на свои):
```
[Unit]
Description=Telegram Event Bot
After=network.target

[Service]
User=www-data
WorkingDirectory=/путь/к/папке/bot
ExecStart=/путь/к/папке/bot/venv/bin/python /путь/к/папке/bot/bot.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

4. Запусти сервис:
```
sudo systemctl daemon-reload
sudo systemctl enable bot.service
sudo systemctl start bot.service
sudo systemctl status bot.service
```


**Вариант 2: Через screen/tmux (простой)**
```
screen -S bot
cd /путь/к/папке/bot
source venv/bin/activate
python bot.py
# Отсоединись: Ctrl+A, затем D
```

### ⚙️ Конфигурация
Все настройки хранятся в файле `.env`:
```
# Токен бота от @BotFather
BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz

# ID администраторов через запятую
ADMIN_IDS=123456789,987654321
```

Как получить:
* *BOT_TOKEN* – отправил в ЛС
* *ADMIN_IDS* – можно узнать у @Get_myidrobot

### 🗄 База данных
Используется *SQLite*, файл `bot_database.db` создаётся автоматически.

Основные таблицы:
* *admins* – администраторы бота
* *groups* – добавленные группы
* *events* – мероприятия
* *registrations* – записи участников

## 🐛 Решение проблем
**Бот не отвечает**
* Проверь, запущен ли процесс
* Проверь токен в .env
* Посмотри логи: journalctl -u bot.service -f

**Бот не видит группу**
* Убедись, что бот добавлен в группу
* Проверь права администратора
* Перезапусти бота

**Ошибка "database is locked":**
* Убедись, что бот запущен в одном экземпляре
